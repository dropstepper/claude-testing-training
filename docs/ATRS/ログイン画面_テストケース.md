# ログイン画面 テストケース

## 設計方針

状態遷移図（ログイン画面_状態遷移図.md）をテストベースとし、
**全状態・全遷移を網羅する**ことを目標に最大5件を設計する。

### カバレッジ設計マップ

| 遷移 | 遷移元 | 遷移先 | カバーするTC |
|---|---|---|---|
| T1 | 入力待ち | クライアント検証 | TC-L-001〜005（全件） |
| T2 | クライアント検証 | クライアントエラー | TC-L-002、TC-L-003 |
| T3 | クライアントエラー | 入力待ち | TC-L-002、TC-L-003 |
| T4 | クライアント検証 | サーバ処理 | TC-L-001、TC-L-004、TC-L-005 |
| T5 | サーバ処理 | 認証エラー | TC-L-004、TC-L-005 |
| T6 | 認証エラー | 入力待ち | TC-L-004、TC-L-005 |
| T7 | サーバ処理 | TOP画面 | TC-L-001 |

**状態カバレッジ：6 / 6（100%）　遷移カバレッジ：7 / 7（100%）**

---

## テストケース一覧

### TC-L-001：正常ログイン

| 項目 | 内容 |
|---|---|
| テストケースID | TC-L-001 |
| テストケース名 | 正常ログイン |
| 遷移パス | 入力待ち → クライアント検証 → サーバ処理 → TOP画面 |
| カバー遷移 | T1、T4、T7 |
| 前提条件 | ATRSが起動しログイン画面が表示されていること／会員番号 `0000000001` がDBに登録済みであること |

**入力値**

| 項目 | 入力値 |
|---|---|
| 会員番号 | `0000000001` |
| パスワード | `aaaaa11111` |

**操作手順**

1. 会員番号に `0000000001` を入力する
2. パスワードに `aaaaa11111` を入力する
3. ログインボタンを押下する

**期待結果**

1. TOP画面が表示される
2. ヘッダにログインユーザ名（姓＋名）が表示される
3. エラーメッセージは表示されない

**入力値**

| 項目 | 入力値 |

### TC-L-003：クライアントエラー（パスワード 未入力）

| 項目 | 内容 |
|---|---|
| テストケースID | TC-L-003 |
| テストケース名 | クライアントエラー（パスワード 未入力） |
| 遷移パス | 入力待ち → クライアント検証 → クライアントエラー → 入力待ち |
| カバー遷移 | T1、T2、T3 |
| 前提条件 | ATRSが起動しログイン画面が表示されていること |

**入力値**

| 項目 | 入力値 |
|---|---|
| 会員番号 | `0000000001` |
| パスワード | （空） |

**操作手順**

1. 会員番号に `0000000001` を入力する
