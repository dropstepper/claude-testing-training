## 1 目的

本マニュアルでは、Macchinetta クライアント版 フレームワーク、Macchinetta オンライン版 フレームワークを利用したサンプルアプリケーション「Airline Ticket Reservation System (以下 ATRS)」の概要、利用方法を説明します。
本マニュアルを読むことで、ATRS の理解が円滑に進むことを目的としています。

## 2 対象読者

本マニュアルは、Macchinetta クライアント版、オンライン版の利用者を対象読者としており、下記の知識、スキルがあることを前提としています。
・ HTML・CSS・JavaScript に関する基礎的な知識がある。
・ jQuery に関する基礎的な知識がある。
・ Java(Servlet/JSP)を使用して Web アプリケーションを開発したことがある。
・ SQL に関する知識がある。
・ Maven による構成管理を行ったことがある。
・ Spring Tool Suite(以下 STS)、または Eclipse を利用してアプリケーションを開発したことがある。

## 5 アプリケーション概要

ATRS の概要、システム構成、利用しているフレームワーク提供機能を説明します。

### 5.1 ATRS 概要

ATRS は、Airline Ticket Reservation System の略称であり、Macchinetta オンライン版 フレームワーク、Macchinetta クライアント版 フレームワークを利用したWeb航空チケット予約システムです。以下の機能を具備しています。
・ システムを利用するための Web ブラウザベースのユーザインターフェース
・ システムへのログイン・ログアウト機能
・ 空席照会・チケット予約機能
・ 会員情報の登録・変更機能

## 6 設定手順

ATRS を動作させるための設定を説明します。「3 動作確認環境」を参照し、事前に環境を準備してください。

### 6.1 Tomcat

Tomcat に対する個別設定はありません。
ATRS はアプリケーションサーバ提供の JDBC データソースを使用しておらず、データソース設定は不要です。

### 6.2 PostgreSQL

PostgreSQL への DB 作成、テーブル作成、テストデータ投入の方法を説明します。
テーブル作成、テストデータ投入には Maven を使用します。

#### 使用ソフトウェア・資材

本手順の使用ソフトウェア・資材を以下に示します。

| 資材 | atrs | C:/atrs |

#### １．ATRSDB の作成

① コマンドプロンプトを起動します。
② 以下を入力し、psql を起動します。
"C:/Program Files/PostgreSQL/13.5/bin/psql" postgres postgres
③ ユーザ postgres のパスワードを要求されますので、パスワードを入力します。
④ psql が起動しますので、以下の SQL を入力し、実行します。

⑤ 「CREATE DATABASE」が表示されることを確認します。
⑥ 「\q」を入力し、psql を終了します。

C:\Users\xx>

#### 補足：テストデータについて

・ピーク時期データは現在年(システム日付)＋来年の 2 年分登録されます。
(ピーク時期はシステムが運賃の算出に利用しています。)
来年以降のデータも投入したい場合は、atrs-initdb の資材を書き換えて「テーブル作成、テストデータ投入」手順を再実施してください。

【対象資材】
atrs-initdb/src/sqls/postgres/00240_insert_peak_time.sql

【変更箇所】
10 行目
addingYear INT := 2;
「2」を投入したい年数分加算した値に変更してください。

・フライト情報データは現在日(システム日付)＋120 日分登録されます。
(システムの仕様としては 90 日先までのフライト情報が照会可能です。)
120 日以降のデータも投入したい場合は、資材を書き換えて「テーブル作成、テストデータ投入」手順を再実施してください。

【対象資材】
atrs-initdb/src/sqls/postgres/00250_insert_flight.sql

【変更箇所】
8 行目
endDate DATE := current_date+120;
「120」を投入したい日数分加算した値に変更してください。
currentDate、endDate に固定値を指定することで、登録期間を明示的に指定することも可能です。

以上で、PostgreSQL の設定は完了です。

## [参考 2]Swagger ツールの使用方法

RESTful Web Service を利用するための Swagger ツールの使用方法を説明します。

### 1. Swagger UI（API ドキュメント生成）の使用方法

ATRS の OpenAPI Specification のファイルから Swagger UI を用いて API ドキュメントを HTML 形式で生成します。

#### 操作手順

① Swagger UI の GitHub ページを開き、releases を開きます。
（https://github.com/swagger-api/swagger-ui）
② Swagger UI のバージョン 3.17.6 をダウンロードします。
③ ダウンロードした Swagger UI を展開します。
④ 展開された swagger-ui/dist の中に「docs」フォルダを作成し、docs フォルダの中に ATRS の OpenAPI Specification のファイルを格納します。

#### 特記事項およびエラー条件

【特記事項】
・HTTP サーバ越しに Swagger UI で OpenAPI Specification を参照するには、dist フォルダをTomcat上に配置する必要があります。（今回は例として Tomcat を使用しています。）

#### 操作手順（続き）

⑤ インストールされた swagger-ui のフォルダ内にある dist フォルダを C:/apache-tomcat-7.0.62/webapps にコピーします。
⑥ C:/apache-tomcat-7.0.62/bin の startup.bat を実行し、Tomcat を起動します。

#### 特記事項およびエラー条件

【特記事項】
・ポート番号は Tomcat で設定されているポート番号を入力します。

#### 操作手順（続き）

⑦ ブラウザを起動し、http://localhost:8080/dist にアクセスする。

#### 特記事項およびエラー条件

【特記事項】
・ポート番号は Tomcat で設定されているポート番号を入力します。
・デフォルトで表示されているのは Swagger が提供しているサンプルの API ドキュメントです。

#### 操作手順（続き）

⑧ アクセスした Swagger UI の画面上部に Tomcat フォルダ内の OpenAPI Specification のファイルパス(http://localhost:8080/dist/docs/atrs-api.json) を指定し、Explore ボタンを押下する。

### 2. OpenAPI Generator（モックサーバの生成・起動）の使用方法

ATRS の OpenAPI Specification のファイルから OpenAPI Generator を用いてモックサーバを生成・起動します。

#### 操作手順

① OpenAPI Generator の GitHub ページを開き、releases を開きます。
（https://github.com/OpenAPITools/openapi-generator）
② OpenAPI Generator のバージョン 3.1.2 をダウンロードします。
③ ダウンロードした OpenAPI Generator を展開します。
④ 展開した OpenAPI Generator のディレクトリ（～/openapi-generator-3.1.2）で以下のコマンドを実行し、ビルドします。

$ mvn clean package

#### 特記事項およびエラー条件

【特記事項】
Spring 形式以外のモックサーバを生成する場合は、コマンド内の 「-g」に任意の形式を指定します。

#### 操作手順（続き）

⑤ モックサーバのソースコードを生成します。
ATRS の OpenAPI Specification ファイルを使用する場合のコマンド例

$ java -jar /c/openapi-generator/modules/openapi-generator-cli/target/openapi-generator-cli.jar generate -i atrs-api.json -g spring -o ./output/
$ java -jar <openapi-generator-cli.jar のパス> generate -i <使用する OpenAPI Specification ファイルのパス> -ｇ spring -o <出力するディレクトリのパス>

⑥ 生成されたソースコードを STS にインポートします。
インポート方法は「6.3 Spring Tool Suite」の「ATRS プロジェクトのインポート」を参照
⑦ 「OpenAPI2SpringBoot.java」を実行します。

#### 特記事項およびエラー条件

【特記事項】
生成されるモックサーバのレスポンスは HTTP ステータスコードがデフォルトで 501 に設定されているため、必要に応じて返すレスポンスを実装する必要があります。

#### 操作手順（続き）

⑧ コンソールビューに「Started～」と表示されることを確認します。
※OpenAPI Generator で生成したソースを Macchinetta に適用する場合は、適宜ソースの修正が必要です。

### 3. OpenAPI Generator（クライアントコードの生成・実行）の使用方法

ATRS の OpenAPI Specification のファイルから OpenAPI Generator を用いてクライアント側の API を実行するソースコードを生成・実行します。

#### 操作手順

① 「2. OpenAPI Generator（モックサーバの生成・起動）の使用方法」の操作手順①～⑤を実行し、OpenAPI Generator の環境を構築します。

#### 特記事項およびエラー条件

【特記事項】
JavaScript 以外のソースコードを生成する場合は、「-g」に任意のソースコードを指定します。

#### 操作手順（続き）

② クライアントのソースコードを生成します。
ATRS の OpenAPI Specification ファイルを使用する場合のコマンド例

$ java -jar /c/openapi-generator/modules/openapi-generator-cli/target/openapi-generator-cli.jar generate -i atrs-api.json -g javascript -o ./output/
$ java -jar <openapi-generator-cli.jar のパス> generate -i <使用する OpenAPI Specification ファイルのパス> -ｇ javascript -o <出力するディレクトリのパス>

#### 特記事項およびエラー条件

【特記事項】
・コマンド実行時、warning が発生しますが問題ありません。
・README.md に記載されている JavaScript は、API コールスクリプトを呼び出し、サーバにリクエストを送信、及びレスポンスを受信するコードとなっています。

#### 操作手順（続き）

③ 生成され たソ ースコードの フ ォルダ内にある README.md の「 Local development」の記載を実行します。
1. 生成されたソースコードのフォルダ(README.md が置かれているディレクトリ)で以下を実行します。

$ npm link

2. README.md の「Getting Started」の項目に記載されている JavaScript のコードを任意のディレクトリに作成します。
3. 2 で作成した JavaScript を保存したディレクトリで以下を実行します。

$ npm link <README.md の置かれたディレクトリのパス>
$ npm install

④ ③で作成した JavaScript を以下のコマンドで実行します。

$ node <作成した JavaScript のファイル名>

Node.js で実行する場合は⑤の手順は不要となります。

⑤ 生成されたソースコードは Node.js 用の関数が使用されているため、ブラウザから API を実行するには、以下の手順を実行します。
1. 「Browserify」をインストールします。

$ npm install browserify@16.2.2

2. 作成した JavaScript ファイルを Browserify でビルドします。

$ node_modules/.bin/browserify <作成した JavaScript のファイル名> -o bundle.js

3. 以下のような html を作成しブラウザで起動することで、JavaScript が実行されリクエストが送信されます。

<!DOCTYPE html>
<html>
<head></head>
<body>
<script type="text/javascript" src="./bundle.js"></script>
</body>
</html>
