## 使用環境の役割分担

| 環境 | 何をするか |
|---|---|
| **AWS（Ubuntu + nginx）** | Linux コマンド練習・Postman の基礎操作 |
| **ローカル ATRS（正常版）** | テストケース検証・Postman 実践・Playwright |

> **ポイント：** AWS は Linux と Postman 入門の「練習台」。本番の学習はすべてローカルの ATRS で行う。

## 進め方

- Claude Code との壁打ちを中心に進める
- AI がドラフトを作り、受講者が直す。このくり返しが核心
- Git はテストケースを作りながら自然に覚える
- メンターが未習熟の技術は先行学習して受講者に展開する

---

## トレーニング内容

### 1. AI を使ったテストケース設計（中心・最優先）

- サンプル仕様書を Claude Code に渡し、テストケースのドラフトを生成させる
- 結果をレビューして修正指示を出す
- 指示の出し方を変えて、結果の違いを体感する

### 2. Git による成果物管理

- テストケースを作るたびに add / commit / push する
- 修正履歴を diff で確認する

### 3. Linux コマンド基礎（AWS Ubuntu）

- SSH で AWS の Ubuntu サーバーに接続する
- `ls` / `cd` / `mkdir` / `grep` / `cat` などの基本コマンドを操作する
- nginx の稼働確認など、サーバー管理の入口を体験する

### 4. API テスト（Postman）

**Stage 1：AWS nginx を使った基礎練習**
- Postman の画面操作・リクエスト送信・レスポンス確認の基礎を習得する

**Stage 2：ローカル ATRS を使った実践**
- Claude Code と壁打ちしながら、ATRS の API テストケースを設計する
- Postman で実際にリクエストを送り、レスポンスを確認する

### 5. 自動テストの概要理解（Playwright）

- Claude Code が生成した Playwright テストコードを実行して結果を確認する
- E2E 自動テストの流れをつかむ

## 学習フロー

```
Phase 1：基礎知識（JSTQB / ISO 29119）            ← 自習推奨
   ↓
Phase 2：テストケース設計 × Claude Code            ← 最優先
   ↓（並行）
Phase 3：Git によるバージョン管理
   ↓
Phase 4：不具合発見演習（ATRS で実践）
   ↓
Phase 5：Linux コマンド基礎（AWS Ubuntu）
   ↓
Phase 6：Postman API テスト
         Stage 1：AWS nginx で基礎練習
         Stage 2：ローカル ATRS で実践
   ↓
Phase 7：Playwright E2E テスト自動化               ← 余裕があれば
```

## リスク対策

| リスク | 対策 |
|---|---|
| Claude Code の操作でつまずく | エラーも Claude Code に聞く練習として活用 |
| テストケースのレビューができない | メンターが一緒に壁打ちする |
| ISO 29119 の理解が追いつかない | 実務で最低限必要な用語だけに絞る |
| Claude Code の使用量が上限に達する | プランのアップグレードを検討する |
| AWS の課金が発生する | 研修終了後にインスタンスを Terminate（削除）する |
