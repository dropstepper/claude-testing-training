# 課題9 不具合レポート

---

## BUG-001 子供料金が適用されず、大人料金で合計金額が算出される

| 項目 | 内容 |
|------|------|
| **不具合ID** | BUG-001 |
| **タイトル** | 12歳未満の搭乗者に子供料金が適用されず、大人料金で合計金額が算出される |
| **重要度** | A（最高） |
| **ステータス** | オープン |
| **発見日** | 2026-05-14 |
| **発見方法** | Playwright 自動テスト（TC-R-001） |
| **使用環境** | ATRS バグ版 / http://localhost:8083/atrs/ / ブラウザ: Chromium |

---

### 重要度の判定根拠

12歳未満の搭乗者に対して子供割引が適用されない場合、**ユーザーが本来より高い金額で予約を確定させる**ことになる。金銭的な不利益を直接ユーザーに与えるため、最高重要度（A）と判定した。

---

### 再現条件

- ATRS にログイン済みであること（会員番号: 0000000001）
- 搭乗者1（大人）に加えて、12歳未満の搭乗者2を追加すること
- 予約フロー B101 → B202 → B203 を経由すること

---

### 再現手順

1. `http://localhost:8083/atrs/auth/login?form` にアクセスする
2. 会員番号 `0000000001`、パスワード `aaaaa11111` でログインする
3. 空席照会画面（B101）で以下の条件を入力し「照会」ボタンを押す
   - フライト種別：片道
   - 出発地：東京(羽田)
   - 到着地：大阪(伊丹)
   - 搭乗日：本日から7日後
   - 搭乗クラス：一般席
4. 検索結果から空席ありの最初のフライトを選択し「選択フライトを予約」ボタンを押す
5. お客様情報入力画面（B202）で「搭乗者を追加」ボタンを押す
6. 追加された搭乗者2に以下の情報を入力する
   - セイ：ヤマダ
   - メイ：タロウ
   - 年齢：**5**（12歳未満）
   - 性別：男性
7. 「予約確認」ボタンを押す
8. 申し込み内容確認画面（B203）の合計金額を確認する

---

### 発生事象（Actual）

申し込み内容確認画面（B203）に表示された合計金額が、**大人運賃 × 2 と同額**になった。

```
大人運賃（1名分）: ¥20,600
合計金額         : ¥41,200  ← 大人運賃 × 2 と一致
```

Playwright アサーション結果：

```
Error: expect(received).toBeLessThan(expected)
Expected: < 41200
Received:   41200
```

---

### 期待結果（Expected）

仕様「合計金額 = 運賃 × 搭乗者数（12歳以上）+（基本運賃 × 60% − 運賃種別ごとの割引額）× 搭乗者数（12歳未満）」に従い、12歳未満の搭乗者には子供料金が適用されるため、**合計金額 < 大人運賃 × 2** になること。

```
期待する合計金額: 大人運賃(¥20,600) + 子供料金(< ¥20,600) < ¥41,200
```

---

### 正常版との比較

| 環境 | 合計金額 | 大人運賃 × 2 | 結果 |
|------|---------|------------|------|
| 正常版 Port:8082 | ¥33,000 | ¥41,200 | ✅ PASS（子供割引が適用されている） |
| バグ版 Port:8083 | ¥41,200 | ¥41,200 | ❌ FAIL（子供割引が適用されていない） |

---

### 影響範囲

- 12歳未満の搭乗者を含むすべての予約
- 子供料金が関係する合計金額の算出処理全般
