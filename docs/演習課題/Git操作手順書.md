# Git 操作手順書

演習課題の成果物を Git で記録し、GitHub に送信するための手順書です。  
課題1ではメンターと一緒に、課題2では自力でこの手順を実行します。

---

## Git とは

**ファイルの変更履歴を記録するツール**です。

テスト条件一覧を作成・修正するたびに「いつ・何を変えたか」が記録され、後から見返したり元の状態に戻したりできます。

| Git がない場合 | Git がある場合 |
|---|---|
| `テスト条件_最終版.md` | `テスト条件.md`（1つのファイル） |
| `テスト条件_最終版2.md` | 変更履歴は Git が管理してくれる |
| `テスト条件_本当の最終版.md` | いつでも過去の状態に戻せる |

テスト成果物を Git で管理すると、「このテストケースはなぜ追加されたのか」「いつ修正されたのか」が追跡できるようになります。現場でも同様の運用が一般的です。

---

## GitHub とは

**Git の記録をクラウドに保存・共有するサービス**です。

Git はあくまで「自分のPC上での記録ツール」です。GitHub に送信（push）することで、他のメンバーと共有したり、PCが壊れてもデータが失われないようにできます。

```
自分の PC                      GitHub（クラウド）
┌──────────────┐              ┌──────────────────┐
│  Git（記録）  │── push ───→ │  リモートリポジトリ │
│              │← pull  ───  │  （共有・バックアップ）│
└──────────────┘              └──────────────────┘
```

| | Git | GitHub |
|---|---|---|
| 何か | 変更履歴を記録するツール | 記録をクラウドで共有するサービス |
| 場所 | 自分のPC内 | インターネット上 |
| オフラインで使えるか | 使える | 使えない |

---

## Git の基本構造

```
① 作業ファイル         ② ステージング           ③ ローカルリポジトリ      ④ リモート（GitHub）
（編集中のファイル）    （commitする準備）         （自分のPCに記録済み）     （クラウドに送信済み）

    ─── git add ──→    ─── git commit ──→        ─── git push ──→
```

| 操作 | 何をするか | GitHub に届くか |
|---|---|---|
| `git add` | ファイルをステージングに追加する | 届かない |
| `git commit` | ステージングの内容をローカルに記録する | 届かない |
| `git push` | ローカルの記録をリモートに送信する | **届く** |

> `commit` だけでは GitHub にはまだ届きません。`push` して初めて反映されます。

## 手順

### 1. ブランチを作成する

作業中のファイルは `main` ブランチに直接置かず、専用のブランチを作って作業します。

**VS Code GUI：**

1. 画面左下のブランチ名（例：`main`）をクリックする
2. 「＋ 新しいブランチを作成...」を選択する
3. ブランチ名を入力して Enter

**ターミナル（CLI）：**

```bash
git checkout -b feature/exercise-results
```

---

### 2. 変更内容を確認する

ステージングする前に、何が変わったかを必ず確認します。

**VS Code GUI：**

1. 左サイドバーの「ソース管理」アイコンをクリックする
2. 「変更」の一覧にファイルが表示されていることを確認する
3. ファイル名をクリックすると差分が右側に表示される

**ターミナル（CLI）：**

```bash
git status        # 変更されたファイルの一覧
git diff          # 具体的な差分の内容
```

---

### 3. ファイルをステージングする

`git add .`（全ファイル一括）は使いません。ファイルを1つずつ指定してステージングします。

**VS Code GUI：**

1. 「変更」の一覧でファイルにカーソルを合わせると `+` ボタンが表示される
2. `+` をクリックして「ステージングされた変更」に移動させる
3. 対象ファイルをすべてステージングする

**ターミナル（CLI）：**

```bash
git add "docs/演習課題/課題1_運賃算出テスト条件抽出.md"
git add "docs/演習課題/課題2_詳細予約規定の決定表テスト.md"
git status   # ステージングされたことを確認する
```

---

### 4. コミットする

コミットメッセージは「何をしたか」が分かるように書きます。

**コミットメッセージの例：**

```
feat: 運賃算出・詳細予約規定のテスト条件一覧を追加

- 課題1：同値分割・境界値分析で運賃算出のテスト条件を抽出
- 課題2：決定表テストで詳細予約規定のテスト条件を作成
```

| 良い例 | 悪い例 |
|---|---|
| `feat: 運賃算出のテスト条件一覧を追加（同値分割・境界値）` | `課題1完了` |
| `fix: ピーク期間の境界値が漏れていたため追記` | `修正` |

**VS Code GUI：**

1. 「ソース管理」パネル上部のテキストボックスにコミットメッセージを入力する
2. `✓` ボタン（「コミット」）をクリックする

**ターミナル（CLI）：**

```bash
git commit -m "feat: 運賃算出・詳細予約規定のテスト条件一覧を追加"

# コミットされたことを確認する
git log --oneline
```

---

### 5. リモート（GitHub）にプッシュする

コミット直後、GitHub をブラウザで確認してみましょう。  
まだファイルが届いていないことを確認してからプッシュします。

**VS Code GUI：**

1. 「ソース管理」パネルの `...` メニューをクリックする
2. 「プッシュ」を選択する
3. 「ブランチを発行しますか？」が表示されたら「OK」をクリックする

**ターミナル（CLI）：**

```bash
# 初回は -u オプションをつける
git push -u origin feature/exercise-results
```

プッシュ後、GitHub をブラウザで再読み込みして、ファイルが反映されていることを確認します。

---

## コマンドチートシート

```bash
git status                          # 現在の状態を確認する
git diff                            # 差分を確認する
git checkout -b <ブランチ名>         # ブランチを作成して切り替える
git add <ファイルパス>               # ファイルをステージングする
git commit -m "<メッセージ>"         # コミットする
git log --oneline                   # コミット履歴を確認する
git push -u origin <ブランチ名>      # リモートにプッシュする（初回）
git push                            # リモートにプッシュする（2回目以降）
```
